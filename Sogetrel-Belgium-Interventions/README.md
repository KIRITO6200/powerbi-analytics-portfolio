# 🛠 Sogetrel Belgium – Intervention Analytics (PDF → Power BI)

## 🎯 Business Problem
Sogetrel Belgium had **no structured tracking system** for field interventions.

The only available data source consisted of **PDF intervention reports**
generated manually by technicians after each client visit.

This caused:
- No reliable monitoring of interventions
- No visibility on delays, overruns, or cancellations
- No KPI tracking for technicians or operations
- No historical analytics

---

## ✅ Solution Delivered
I designed and implemented an **end-to-end analytics pipeline**:

1. **Python ETL pipeline**
   - Parse unstructured PDF reports
   - Extract operational, temporal, technical, and quality data
   - Normalize and consolidate into a structured dataset (Excel)

2. **Data modeling & analytics**
   - Clean and enrich extracted data
   - Compute operational KPIs (realization rate, delays, overruns, absences)
   - Prepare Power BI–ready tables

3. **Power BI dashboards**
   - Intervention performance
   - Technician performance
   - Cancellation and client absence analysis
   - Geographic distribution and trends

---

## 🛠 Tools & Technologies
- Python (PDF parsing, regex, ETL)
- PyMuPDF (PDF extraction)
- Pandas
- Power BI
- DAX
- Operational analytics & KPI design

---

## 🧠 Key Value for the Client
- Transformed **unstructured PDFs** into **actionable data**
- Enabled **daily reporting & historical analysis**
- Provided **first-ever KPIs** for field interventions
- Supported operational and managerial decision-making

⚠️ All data displayed is anonymized.
